# Web assets for project

## Cách dùng nhanh với GitHub Copilot
- Chép nguyên thư mục `web_assets/images` vào project của bạn, ví dụ: `public/assets/images/`.
- Trong code, dùng các path tương đối như bên dưới.
- Mỗi ảnh đã có `alt`, `section dùng phù hợp`, và `nguồn gốc` để dễ map vào UI.

## Gợi ý tách riêng web và thuyết trình
- **Web**: dùng như một web tài liệu / web trải nghiệm về nội dung chương học.
- **Phần thuyết trình**: khi lên lớp, tập trung nói quanh **Khu 3 – Chủ đề tranh luận** và chỉ dùng web như nền trực quan / tài liệu mở rộng, không đi kiểu click tour qua mọi trang.
- Nghĩa là: web là sản phẩm chính, còn phần nói là một bài tranh luận học thuật về đề tài.

## Gợi ý import / path cho Copilot

### Nếu dùng Next.js với thư mục `public`
```tsx
<Image
  src="/assets/images/quochoi_vietnam_2018.jpg"
  alt="Tòa nhà Quốc hội Việt Nam nhìn từ phía trước"
  width={1600}
  height={900}
  className="object-cover w-full h-full"
/>
```

### Nếu dùng React Vite với thư mục `public`
```tsx
<img
  src="/assets/images/book_nguyen_phu_trong_cover.jpg"
  alt="Bìa sách của Tổng Bí thư Nguyễn Phú Trọng về phòng chống tham nhũng, tiêu cực"
  className="w-full h-auto rounded-2xl"
/>
```

### Nếu muốn dùng background image
```tsx
<section
  style={{ backgroundImage: "url('/assets/images/quochoi_vietnam_2018.jpg')" }}
  className="bg-cover bg-center"
>
```

## Gợi ý viết lại phần thuyết trình cho đúng ý bạn
- **Không thuyết trình bằng cách đi tour toàn web**.
- Khi lên lớp, chỉ dùng web theo kiểu:
  1. mở Trang chủ trong 20–30 giây để giới thiệu sản phẩm;
  2. lướt nhanh Khu 1 để chứng minh bám chương học;
  3. vào thẳng **Khu 3 – Chủ đề tranh luận** và nói toàn bộ lập luận ở đó;
  4. cuối cùng quay sang **Khu 4 – Công dân và tình huống** để chốt trách nhiệm công dân.
- Nghĩa là: **web là tài liệu / trải nghiệm**, còn **bài thuyết trình là bài tranh luận học thuật** nằm bên trong web.
